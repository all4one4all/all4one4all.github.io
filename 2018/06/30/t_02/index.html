Objects And Instances 对象及实体

对象几乎在游戏中每一处都会用到，从游戏中的角色到敌人，从子弹到子弹的爆炸。您将了解到通过代码如何使对象运转及作出反应。但是，在创建第一个对象之前，我们先了解一下对象的概念。

对象背后思想其实是创建一个在游戏中使用的动作蓝图。我们将对象视作一个蓝图是因为实际游戏中没有任何的对象，而只存在着 instances（实体）。实体是初始化对象的拷贝，并被置于游戏 Rooms（区域块）中。因此，对象只存在于资源树中，之后被用来创建存在于区域块中的实体。这意味着改变一个实体的属性将不会改变创建为这个实体的对象的属性；而改变一个对象的属性将会随之改变所有由这个对象创建的实体的属性。

现在，我们将创建一个对象。这个对象将作为游戏中角色，并且根据操作在区域块中移动、射击。创建对象只需在资源树中的对象上单击鼠标右键并选择创建:
这个操作将创建一个新的空的对象，并打开了 Object Editor（对象编辑器）：
对象编辑器将会像精灵编辑器一样在之前的工作区打开，并且对象编辑器还会将精灵编辑器推出可见区域外。您可以按下鼠标滑轮并移动来定位工作区，使用 Ctrl / Command + 鼠标滑轮来缩放工作区。同时也可以使用 Ctrl / Command + Tab 来打开工作区控制台来切换需要的编辑器。

现在，我们已经创建了对象，之后需要命名。按照之前同样的命名法则，使用一个可以定义其资源类型的前缀加上一个独一无二的名称。所以为了匹配之前精灵的命名“spr_player”，我们将这个对象命名为“obj_player”。

现在需要通过在可用精灵列表中选择，来将精灵关联到这个对象。您需要点击当前标记为“No Sprite”的按钮，之后在可用精灵列表中选择名为“spr_player”的精灵：
将一个对象与一个精灵关联意味着当您在游戏区域块中放置这个对象的实体时，精灵的图像及某些属性将会被赋予这个实体。接着让我们在游戏区域块中放置实体吧…

Rooms 区域块

当您创建一个新的项目时，唯一被默认创建的资源便是 Room Resource(区域块资源)。这是因为所有的游戏都需要最少一个区域块来运行，所以 GameMaker Studio 2 会默认创建区域块资源。您可以删除、编辑或者在资源树中的区域块上单击鼠标右键选择创建，本教材中我们将编辑这个名为“room0”的区域块
在资源树中双击这个区域块，将在一个单独的工作区打开 Room Editor(区域块编辑器)：
区域块在游戏的任何部分都会被运用到。您可以为游戏的每一个部分单独运用一个区域块，比如启动页面、主菜单、引导页面及不同关卡等等；或者只运用一个区域块，而其它部分都用代码来生成。一般来说，需要将这两种方法搭配使用，一些区域块来满足多种功能（比如将整个游戏中所有菜单运用于一个区域块），而其他区域块只满足单一的功能，比如俯视地图区域块或者关卡区域块。

当区域块编辑器打开后，您可以看到大部分屏幕都被一个带有网格的黑色区域占据。这个区域即实际的区域块，一个可以放置资源来创建游戏关卡的空白画布。

所有需要放入区域块的资源都会被置于 layer(图层)上（本教材之后的章节会详细阐述图层的概念）。所以，一定要保证在图层属性窗口中选中 Instance Layer(实体图层)：
现在，需要将之前的“obj_palyer”实体拖拽进区域块中。使用鼠标左键选中资源树中的对象，然后拖动到区域块中。
现在我们已经有了区域块和对象，可以实际运行起游戏。点击 Run Button 即编辑器左上角的小三角按钮。您将会看到一些文字在窗口的底部滚动，之后游戏就开始运行了。

Events And Code 事件及代码

现在，您可以关闭游戏运行窗口，然后双击技能树中名为“obj_player”的对象。这将返回主工作区并聚焦于我们需要编辑的对象。
您会注意到有另一个窗口与对象编辑器链接在一起，这个窗口即 Event Editor (事件编辑器)。一个 GameMaker Studio 2 中的游戏仅仅是一个事件中的代码被每秒多次执行的选择。这被称为 Game Loop (游戏主循环)，一个单一的游戏循环被称为 Game Frame(游戏帧)。每一秒的游戏帧数即游戏的 Frames Per Seconds (FPS)值。一般FPS默认为30帧，这表示游戏中的游戏帧(循环遍历所有游戏代码)每秒将会被执行30次，在游戏代码的每一帧中都有事件。这些事件被分为两大类：

  • 一些事件发生在每一个游戏循环中，像Draw event(绘制事件)或者Step event (步行事件)
  • 一些事件只在满足一定条件时才发生，像Keypress event(键盘事件)或者Create event (创作事件)

您可以点击Add Event来查看所有的时间类型：

在上图中您可以看到”Step”类以及其中的 Step Event 处于高亮状态，这就是我们需要向对象中添加的事件。当您选择了Step后，将看到事件添加到事件编辑器窗口中，并链接了一个新的窗口：

这个新的窗口就是DnD Editor(拖放编辑器)，拖放编辑器被分成了三部分：

  • 左边为action list(动作列表)——这是一个事件中所有动作的概览列表
  • 中间为action workspace(动作工作区)——这是放置需要执行并编辑的动作的工作区
  • 右边为action toolbox(动作工具箱)——在这能找到创建游戏所需的拖放动作块

拖拽的想法很简单——从工具箱中选择一个动作，然后把它拖到动作工作区。在工作区中编辑需要的参数，然后继续拖拽下一个动作，并将这个动作与之前的动作链接起来。请注意，动作是以线性方式添加的，因此每个后续动作都将添加在前面动作添加之后。让我们添加一个动作来看看实际是如何工作的……

Movement 运动

现在我们需要让角色实体在用户按下键盘上的方向键时移动。目前，我们在每个游戏帧的 x 位置值上都增加了4，我们需要让键盘右方向键被按下时才触发这个动作。为此，我们使用if key down动作，如果给定的键被按住，那么它会判定为 true 或是为 false ，如果它的值为true，那么我们将把实体的位置值加 4。您可以在动作工具箱的鼠标和键盘部分找到这个动作，然后需要将其拖放到动作工作区中的赋值变量动作上面:

    Rotation 转向

    我们差不多完成了这个教程的第一章，现在我们需要让角色实体完成最后一件事情…当角色实体在屏幕上运动时我们需要让它始终朝向于鼠标方向。 这样，玩家可以用键盘移动并用鼠标瞄准和射击。 我们将射击放在教程的下一章节，现在只需添加转向。
    为此，我们将使用所有对象都拥有的另一个内置变量 image_angle。 此变量控制精灵及赋值实体的角度，因此将其设置为0以外的任何值都将使精灵（及其关联的碰撞蒙版）旋转给定的值。 默认情况下，对于区域块中的每个实体，此值都设置为0，但可以使用Set Instance Rotation动作随时更改它。
