Chapter1 - Movement 移动

开始一个新的项目

这是教程My First Game - DnD的第一个章节，本教程中，我们使用GameMaker Studio 2构建你的第一个俯视视角小游戏——arena shooter（竞技场射手），游戏中会包含一个玩家，多个敌人以及大量子弹，游戏的最终效果如下：
我们尽可能的简化这个教程，为此我们将提供足够的概述，来帮助你摆脱困境并快速完成第一个属于你自己的游戏。
当你打开教程时，将会自动新建一个Drag and Drop项目，你可以使用这个项目来继续完成你的tutorial game（教程游戏）。接下来，我们会讲解如何从零开始创建一个项目。

首先，打开GameMaker Studio 2，然后点击开始页面上的新建项目按钮：
之后会有两个选项：

  • 新建Drag and Drop项目
  • 新建GameMaker Language项目

Drag and Drop（D&D）是一种十分强大的可视化脚本语言，允许你使用链接在一起的action blocks（动作砖块）来完成项目代码
GameMaker Language（GML）则是正式的脚本语言，使用代码编辑器来创建游戏。
这份教程主要讲解 D&D ，所以选择Drag and Drop项目按钮，并为新建的项目命名

注意：如果你拥有一定的编程经验或者对可视化毫无兴趣，对你来说使用 GML 来代替 D&D 可能是更好的选择，并且本教程的新建项目步骤同样适用于 GML 代码编程

当您创建并保存了初始空白项目，将会看到GameMaker Studio 2的workspace（工作区）

工作区

窗口中间的主区域就是工作区，您将在其中完成大部分工作。工作区也有不同的类型，稍后我们将详细介绍。在工作区右边，是 Resource Tree（资源树），它列出了项目可以使用的所有不同元素，比如Sprites（精灵）、Objects（对象）或Rooms（平台）。在本教程中，我们将在使用时解释各种资源的用法，但并不会涵盖所有类型。
特别强调，资源树栏一般停靠在窗口的右侧，可以使用dock按钮打开或关闭资源树:

同时，资源树也可以移动到屏幕的不同位置并停靠在那里:

GameMaker Studio 2支持自定义，因此您可以更改颜色、字体、甚至设置窗口布局，并保存这些设置。本教程将不讨论该程序的自定义部分，而会使用默认布局，但是您可以在用户手册中找到关于自定义的更多信息。（按 F1 获取更多帮助）

精灵

现在，我们有了新的项目，空的资源数，是时候开始构建游戏了。本小节将会在屏幕中创建一个简单的角色对象，并使其移动，此时就需要用到精灵。精灵是一种附带某些属性的图像，这些属性在游戏中被用来表示某些事物。
此时，我们要创建一个精灵来表示我们的角色。

创建一个精灵。只需在资源树中的精灵上单击鼠标右键并选择创建:

这个操作将打开Sprite Editor（精灵编辑器），我们可以在其中添加图像并为其设置属性:

首先，需要为这个精灵命名。这个名字将会是这个精灵在游戏里唯一标识符，可以以任何字符为其命名，但是我们建议使用某种类型的前缀如”spr“或更加简单的”s“，当你需要在游戏代码中创建更加复杂的资源类型时，这种前缀命名的方式会使代码可读性更强，在此，我们将当前这个精灵命名为”spr_player“。

我们需要为这个精灵添加一个图像作为我们角色的外观，本教程中，我们将使用下面的图片作为精灵的图像，你也可以自行选择图片：

你需要点击Import按钮打开文件夹选择合适的图片，图片格式必须为 PNG 、 GIF 或者 JPG
本教程中，已在项目内创建了一个名为 TutorialResources的文件夹，其中包含本教程用到的图片及音频资源，在子文件夹Images中可以找到所需图像。

当您打图像时，会提示“本操作不可撤销”，不用担心，因为我们正在创建一个新的精灵，而不是覆盖一个原有的精灵。所以，点击Yes继续。此时，精灵编辑器会如下图所示：

编辑器的顶部会展示一个单独的图像，而主窗口则是这个单独图像的预览。如果添加的是动画，顶部则会展示动画的关键帧，点击这些关键帧，同样可以在主窗口预览。如果添加的是单张图像，则主窗口显示这张图片的预览。

现在，我们已经添加了图像，现在我们需要设置图像的origin（原点）。原点是精灵上一个被用来在Rooms（平台）上确定其位置的点，你可以在预览图像中任何位置点击鼠标左键来确定这个精灵的原点，或者在下拉菜单中设置一个固定位置为圆点：

在上图中，我们将这个精灵的原点设置为Middle Center(正中)。当这些操作完成后，我们就可以准备使用我们的精灵创建一个Object Resource（对象资源）。

对象及实体

对象几乎在游戏中每一处都会用到，从游戏中的角色到敌人，从子弹到子弹的爆炸。您将了解到通过代码如何使对象运转及作出反应。但是，在创建第一个对象之前，我们先了解一下对象的概念。

对象背后思想其实是创建一个在游戏中使用的动作蓝图。我们将对象视作一个蓝图是因为实际游戏中没有任何的对象，而只存在着instances（实体）。实体是初始化对象的拷贝，并被置于游戏Rooms（平台）中。因此，对象只存在于资源树中，之后被用来创建存在于游戏平台中的实体。这意味着改变一个实体的属性将不会改变创建为这个实体的对象的属性；而改变一个对象的属性将会随之改变所有由这个对象创建的实体的属性。

现在，我们将创建一个对象。这个对象将作为游戏中角色，并且根据操作在游戏平台中移动、射击。创建对象只需在资源树中的对象上单击鼠标右键并选择创建:

这个操作将创建一个新的空的对象，并打开了Object Editor（对象编辑器）：

对象编辑器将会像精灵编辑器一样在之前的工作区打开，并且对象编辑器还会将精灵编辑器推出可见区域外。您可以按下鼠标滑轮并移动来定位工作区，同时也可以使用 Ctrl / Command + 鼠标滑轮来缩放工作区。同时也可以使用 Ctrl / Command + Tab 来打开工作区控制台来切换需要的编辑器。

现在，我们已经创建了对象，之后需要命名。按照之前同样的命名法则，使用一个可以定义其资源类型的前缀加上一个独一无二的名称。所以为了匹配之前精灵的命名“spr_player”，我们将这个对象命名为“obj_player”。

现在需要通过在可用精灵列表中选择来将精灵关联到这个对象。您需要点击当前标记为“No Sprite”的按钮，之后在可用精灵列表中选择名为“spr_player”的精灵：

将一个对象与一个精灵关联意味着当您在游戏平台放置这个对象的实体时，精灵的图像及某些属性将会被赋予这个实体。让我们接下来在游戏平台放置实体吧…
