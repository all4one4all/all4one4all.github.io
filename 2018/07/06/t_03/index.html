Chapter2 - Shooting 射击

Bullets 子弹

首先回顾一下上一章的内容。 在上一章中，我们创建了一个精灵，由这个精灵创建一个对象并将该对象的实体放置于区域块中。 然后，使其能依靠方向键来移动，并始终朝向于鼠标方向。我们开了一个好头，但它仍然算不上是一个游戏，所以在本章中我们将为项目添加一个射击机制。
很明显，我们首先需要一个新的精灵来表示子弹，在教程的第一章中已经详细介绍了如何创建精灵，在此简略叙述流程：

  • 在资源树中的Sprites上单击鼠标右键
  • 选择Create创建一个新的精灵
  • 把这个精灵命名为 “spr_bullet”
  • 在Import上单击鼠标左键，并选择一个适合的图像
  • 设置精灵的原点

如果您创建的精灵使用了本教材提供的图像，最好将精灵的原点设置在子弹图像的头部附近（如下图所示），因为这是我们想要旋转它并将其“固定”入区域块的点。 如果您使用了自己的精灵，请单击您认为最适合的位置来设置图像原点。

The Bullets Object 子弹对象

如同创建精灵，在教程的第一章中也同样详细介绍了如何创建对象，在此简略叙述流程：

  • 在资源树中的Objects上单击鼠标右键
  • 选择Create创建一个新的对象
  • 把这个对象命名为 “obj_bullet”
  • 在No Sprite上单机鼠标左键，选择我们之前创建的精灵

新创建的对象已经链接了一个事件窗口方便我们添加一些代码，但是首先我们需要返回之前的角色对象，做一些小的修改。在技能树中的”obj_palyer”对象上双击鼠标左键在当前工作区打开。

Layers 图层

在开始之前，我们应该先深入了解一下层的概念。 我们使用Instance Create动作创建了子弹实体，这个动作会创建实体并且把它分配到给定的图层ID上——在本教程中为实体的默认区域块编辑器图层。但什么是图层呢？简单来说，图层是我们可以用来存储内容的2D空间，区域块编辑器中图层的顺序将影响绘制实体的深度：

在上图中，您可以看到通过更改图层顺序，我们可以更改是否在敌人实体上绘制了玩家实体。 请注意，如果在同一图层上有多个实例，那么这些实体将以最直接的方式绘制，通常从该图层上创建的第一个实体到最后一个，但这不能保证呈现出您所需要的效果。 因此，如果希望某些内容出现在其他内容之上或之下，则应该改变其图层位置。 下图是一个可视化的图层渲染示意图来帮助您更好了解这一过程：

在我们的教程游戏中，通过使用区域块编辑器中名为“Instances”的图层，我们将子弹实体与“obj_player”实体分配到了相同图层。 所有对象都必须放置在图层上，当在区域块中创建实体时，必须将其分配给图层，并且这个图层将影响渲染顺序，即：在其他图层的内容“上方”还是“下方”来绘制。 例如，我们可以在区域块编辑器中创建一个“BulletsLayer”，然后明确地表示子弹实体将被创建在另一个图层上——如下图所示：

可以看到，如果在房间编辑器中创建了一个图层，我们可以使用其名称作为标识符来在动作中定位它。 我们稍后将在本教程中执行此操作，只需使用“Instances”层开始。
如果要了解有关图层功能的更多信息，请按 F1 打开用户手册并搜索“图层”。
